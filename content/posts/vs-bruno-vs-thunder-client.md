# 一个代码编辑器，凭什么让开发者吵翻天？

去年底，VS Code 的官方插件市场里，REST Client 类插件的下载量突破了 2000 万次。其中两个名字最扎眼：Bruno 和 Thunder Client。

一个主打“纯本地、无云端”，另一个标榜“轻量级、全功能”。开发者们分成了两派，在 Reddit 和 GitHub 上吵得不可开交。有人说 Bruno 是 Postman 的完美替代，有人觉得 Thunder Client 才是 VS Code 里的真香选择。

**说白了，这俩工具解决的问题一模一样：帮开发者调试 API。** 但实现路径和哲学，差了十万八千里。

---

## Bruno：一个“反云端”的异类

Bruno 的创始人 Anoop 在 2023 年接受采访时说过一句话：“我们讨厌 SaaS。” 这不是营销话术，而是产品的核心逻辑。

打开 Bruno，你会发现它根本没有注册流程。所有数据都存成纯文本文件，放在你本地的文件夹里。想分享？直接把文件丢进 Git 仓库就行。**据 Bruno 官方数据，2024 年初其 GitHub Star 数突破 1.5 万，而它的安装包大小只有 30MB 左右。** 对比 Postman 的 200MB+，这个体积确实让人舒服。

但代价也很明显。不支持团队协作，没有云端同步，不保存历史请求记录。如果你是个独立开发者，或者团队有严格的离线开发要求，Bruno 会非常顺手。但如果你需要和同事共享环境变量、对比历史版本，Bruno 会让你抓狂。

**说白了，Bruno 赌的是“本地优先”这个趋势。** 它赌开发者厌倦了 SaaS 的订阅制、隐私泄露和臃肿界面。这个赌注目前看对了，但前提是你得忍受它“单机版”的孤独感。

---

## Thunder Client：藏在 VS Code 里的“瑞士军刀”

Thunder Client 走的是另一条路。它完全寄生在 VS Code 里，安装后直接在侧边栏出现，不需要单独开窗口。**根据其开发者 Ranga 在 2023 年底的博客，Thunder Client 的周活跃用户超过 80 万。** 这个数字在 VS Code 插件生态里排进前 5%。

它的优势是“无缝”。写代码时按 Ctrl+Shift+P 就能调出请求面板，响应速度比 Postman 快得多。支持 GraphQL、环境变量、脚本预执行，甚至能生成代码片段。**一个细节：Thunder Client 的脚本引擎用的是 JavaScript，而 Bruno 用的是自家定义的 DSL（领域特定语言）。** 这意味着你在 Thunder Client 里写脚本，可以直接复用写业务逻辑时的 JS 知识。

但缺点也很刺眼。它依赖 VS Code 的运行环境，如果 VS Code 崩溃，你的请求数据可能丢失。而且它的界面在深色主题下经常出现字体渲染问题，一些用户反馈“看着眼睛疼”。

**说真的，Thunder Client 更适合“轻量级”场景。** 快速测试一个接口、调试一次登录流程，它比 Bruno 快。但如果你要管理上百个接口的环境变量、做复杂的断言测试，它的稳定性会让人心里没底。

---

## 到底选哪个？看你的工作流

别纠结谁更好，先问自己三个问题。

**第一，你的数据敏感吗？** 如果你的 API 请求里包含客户密钥或内部系统地址，Bruno 的纯本地存储是唯一安全的选择。Thunder Client 虽然也存本地，但它的配置文件在 VS Code 的全局目录里，容易被误删。

**第二，你有多依赖 Git？** Bruno 直接把请求文件当成 Git 管理的代码文件，你可以用 diff 工具对比两个版本。Thunder Client 的数据是 JSON 文件，虽然也能 Git 管理，但冲突处理非常麻烦。

**第三，你愿意多开一个窗口吗？** 如果你习惯在 VS Code 里完成所有操作，Thunder Client 的“内嵌式”体验是杀手锏。Bruno 需要单独启动一个进程，虽然它强调“轻量”，但多一个窗口就多一分分心。

**一个有意思的数据：据 JetBrains 2023 年开发者调查，42% 的受访者使用 Postman，而使用 VS Code 内置工具的比例只有 18%。** 但 Bruno 和 Thunder Client 的增长速度正在改写这个格局。它们不是要取代 Postman，而是给开发者多了一个“去中心化”的选择。

---

## 没有完美工具，只有适配场景

Bruno 和 Thunder Client 的争论，本质上是“本地优先”和“集成体验”的路线之争。没有谁对谁错。

如果你是个独立开发者，或者团队规模小于 5 人，Bruno 的干净和透明会让你上瘾。如果你在大型企业，需要和十几个人共享环境变量，Thunder Client 的 VS Code 集成会让你少挨骂。

**最后一句：别被“开源”和“免费”这些标签绑架。** 工具是拿来用的，不是拿来信仰的。打开 VS Code，装两个插件，各跑 10 分钟。哪个让你觉得“卧槽，真顺”，就用哪个。其他都是扯淡。
