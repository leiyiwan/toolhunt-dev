# Postman vs Bruno：开发者到底该选哪个 API 客户端？

凌晨两点，程序员小王盯着 Postman 的界面发呆。他的团队刚收到 Postman 的新版定价邮件：免费版限制 3 个协作成员，高级版每人每月 12 美元。他算了一笔账，5 人团队一年要花掉 720 美元。这不是小数目。

与此同时，一个叫 Bruno 的开源工具正在开发者社区悄悄扩散。GitHub 上它已经拿到 8000 多颗星，很多人说它是 Postman 的替代品。但真能替代吗？我们来拆开看看。

## 核心差异：云端 vs 本地

Postman 是一个云端优先的工具。你的 API 请求、环境变量、测试脚本都存储在 Postman 的服务器上。这意味着你要联网才能用，数据也托管在别人那里。

Bruno 反过来。它把所有数据存成纯文本文件，就在你的项目仓库里。你可以用 Git 管理这些文件，团队成员 pull 一下就能同步。离线也能运行，数据永远在自己手里。

这种差异决定了后续的一切。Postman 方便协作，但依赖网络和平台。Bruno 更自主，但需要团队自己维护版本控制。

## 协作模式：谁更适合团队？

Postman 的协作很成熟。你创建一个工作区，邀请成员，大家共享集合和环境。还能评论、分享文档。但代价是，免费版只能 3 个人协作。超过就要付费。

Bruno 的协作走的是另一条路。因为文件是文本格式，你可以直接用 Git 分支管理。一个人改了一个接口，其他人 pull 下来就能看到。没有邀请机制，没有权限控制，就是最朴素的代码协作。

说实话，这不是谁好谁坏的问题。如果你的团队习惯 Git 工作流，Bruno 会让你觉得自然。如果你需要可视化协作和权限管理，Postman 更顺手。

## 功能对比：谁更能打？

**请求构造**  
两者都支持 GET、POST、PUT、DELETE 等常见方法。Postman 的 UI 更精致，有自动补全和历史记录。Bruno 的界面简洁但够用，参数表格、请求头、Body 都能改。

**环境变量**  
Postman 支持多环境切换，变量作用域细到集合、环境、全局。Bruno 同样支持，但通过 `.env` 文件管理，更接近开发环境习惯。

**测试脚本**  
Postman 用 JavaScript 写测试，支持断言、数据驱动、Runner 批量运行。Bruno 也支持 JavaScript 脚本，但生态不如 Postman 丰富。据 Postman 官方数据，其测试脚本库有超过 1000 个社区贡献的代码片段。

**文档生成**  
Postman 一键生成 API 文档，还能发布到网页。Bruno 没有这个功能，你得自己写文档或用其他工具。

## 定价：钱的问题很现实

Postman 的免费版够个人用，团队用就受限。Pro 版每人每月 12 美元，Enterprise 版要 20 美元。按 10 人团队算，一年 1440 到 2400 美元。

Bruno 完全开源免费。没有隐藏收费，没有功能限制。据其官网，它采用 MIT 许可证，你可以修改、分发、商用。

但免费也有代价。Bruno 没有官方支持，社区也小得多。如果你遇到 Bug，得自己去 GitHub 提 Issue 或者翻代码。

## 场景选择：什么时候用哪个？

**选 Postman 的场景**  
- 团队不熟悉 Git，需要可视化协作  
- 需要自动生成 API 文档并分享  
- 使用 Postman 的 Mock Server 和监控功能  
- 客户或非技术人员需要查看接口

**选 Bruno 的场景**  
- 团队使用 Git 管理一切  
- 对数据安全性有要求  
- 预算有限，不愿为协作付费  
- 喜欢离线工作，不依赖网络

**两者都不选的场景**  
- 只做简单测试，用 curl 就行  
- 需要大量自动化测试，直接上 JMeter 或 K6

## 一点个人看法

Postman 和 Bruno 不是替代关系，是两种不同的哲学。Postman 像一个功能齐全的 SaaS 产品，什么都给你配好。Bruno 像一个开源工具箱，你自己搭积木。

如果你问我，我会说：小团队、安全敏感的项目、习惯 Git 的人，可以试试 Bruno。大团队、需要可视化协作、愿意付费的人，继续用 Postman 没问题。

最后说一句，工具是手段，不是目的。别在选工具上花太多时间，写代码才是正事。
